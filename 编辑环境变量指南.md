# 如何编辑环境变量确保使用 Python 3.10

## 方法一：图形界面手动编辑（推荐）

### 步骤 1：打开环境变量设置

1. 按 `Win + X` 键，选择"系统"
2. 或者：右键点击"此电脑" → 选择"属性"
3. 点击"高级系统设置"
4. 在"系统属性"窗口中，点击"环境变量"按钮

### 步骤 2：编辑系统 PATH（需要管理员权限）

**⚠️ 重要：修改系统 PATH 需要管理员权限**

1. 在"环境变量"窗口中，找到"系统变量"区域
2. 找到名为 `Path` 的变量，选中后点击"编辑"
3. 查找并**删除**以下路径（如果存在）：
   - `C:\Python314\`
   - `C:\Python314\Scripts\`
4. 点击"确定"保存

### 步骤 3：编辑用户 PATH（推荐，不需要管理员权限）

1. 在"环境变量"窗口中，找到"用户变量"区域
2. 找到名为 `Path` 的变量，选中后点击"编辑"
3. 确保以下路径**在最前面**（使用"上移"按钮）：
   - `C:\Users\hlin2\AppData\Local\Programs\Python\Python310`
   - `C:\Users\hlin2\AppData\Local\Programs\Python\Python310\Scripts`
4. 如果这些路径不存在，点击"新建"添加它们
5. 点击"确定"保存

### 步骤 4：验证

1. **关闭所有已打开的终端/命令提示符窗口**
2. 打开新的 PowerShell 或命令提示符
3. 运行以下命令验证：

```powershell
python --version    # 应该显示 Python 3.10.11
where.exe python    # 第一个路径应该是 Python 3.10
```

## 方法二：使用 PowerShell 脚本自动修改

### 快速方法（推荐）

1. **以管理员身份运行 PowerShell**：
   - 按 `Win + X`
   - 选择"Windows PowerShell (管理员)" 或 "终端 (管理员)"
   - 或者在开始菜单搜索"PowerShell"，右键选择"以管理员身份运行"

2. 导航到项目目录：
```powershell
cd C:\Users\hlin2\jewelry-ai-erp
```

3. 运行脚本：
```powershell
.\set_python310_default.ps1
```

4. 脚本会自动：
   - 从系统 PATH 中移除 Python 3.14
   - 确保用户 PATH 中 Python 3.10 在最前面

5. **关闭所有终端窗口并重新打开**，然后验证：
```powershell
python --version
```

## 方法三：仅修改用户 PATH（不需要管理员权限）

如果你不想修改系统 PATH，可以只确保用户 PATH 中 Python 3.10 在最前面：

### 使用 PowerShell（当前用户）

```powershell
# 获取当前用户 PATH
$userPath = [Environment]::GetEnvironmentVariable('Path', 'User')
$paths = $userPath -split ';' | Where-Object { $_ }

# Python 3.10 路径
$python310Paths = @(
    'C:\Users\hlin2\AppData\Local\Programs\Python\Python310',
    'C:\Users\hlin2\AppData\Local\Programs\Python\Python310\Scripts'
)

# 移除重复的 Python 3.10 路径，然后重新排列
$otherPaths = $paths | Where-Object { $_ -notin $python310Paths }
$newUserPath = ($python310Paths + $otherPaths) -join ';'

# 保存
[Environment]::SetEnvironmentVariable('Path', $newUserPath, 'User')

Write-Host "用户 PATH 已更新！请关闭并重新打开终端。" -ForegroundColor Green
```

## 当前环境变量状态

### 系统 PATH 中的 Python 路径：
- `C:\Python314\Scripts\`
- `C:\Python314\`
- `C:\Program Files\Python312\Scripts\`
- `C:\Program Files\Python312\`

### 用户 PATH 中的 Python 路径：
- `C:\Users\hlin2\AppData\Local\Programs\Python\Python310`
- `C:\Users\hlin2\AppData\Local\Programs\Python\Python310\Scripts`

**问题**：系统 PATH 中的 Python 3.14 优先级高于用户 PATH，所以 `python` 命令会先找到 Python 3.14。

**解决方案**：从系统 PATH 中移除 Python 3.14，或确保用户 PATH 中的 Python 3.10 路径在系统 PATH 之前（通过 PowerShell 配置文件已实现）。

## 验证配置是否成功

运行以下命令检查：

```powershell
# 检查 Python 版本
python --version

# 检查 Python 路径（第一个应该是 Python 3.10）
where.exe python

# 检查 py launcher
py --version
py -0  # 查看所有已安装的版本
```

---

## 注意事项

1. **修改环境变量后必须重启终端**才能生效
2. **系统 PATH 修改需要管理员权限**
3. **用户 PATH 修改不需要管理员权限**，但优先级低于系统 PATH
4. 如果使用 PowerShell，配置文件会自动调整 PATH 优先级（已配置）

---

## 推荐方案

**最简单的方法**：使用已创建的 PowerShell 配置文件，它会自动在每次打开 PowerShell 时设置 Python 3.10 为默认。无需修改系统环境变量。

如果需要永久性更改（包括 CMD 命令提示符），则需要：
1. 以管理员身份运行 `set_python310_default.ps1`
2. 或手动编辑系统环境变量（方法一）
