# 珠宝ERP业务规则知识库

> 此文件包含系统的核心业务规则，AI在处理请求时必须遵守这些规则。

## 一、入库规则

### 基本要求
- 入库必须填写：商品名称、克重、工费（元/克）
- 供应商为可选项，但建议填写以便追溯
- 入库单号格式：RK + 日期 + 序号（如 RK20260112001）

### 工费标准
- 普通黄金饰品：工费范围 30-80 元/克
- 镶嵌类饰品：工费范围 80-200 元/克
- 特殊工艺：工费可超过 200 元/克

## 二、销售规则

### 必填信息
- 柜台开单必须填写：客户姓名、业务员、商品名称、克重、工费
- 备注为可选

### 销售单状态
- 待结算：已开单，等待客户付款
- 已结算：已完成支付
- 已取消：作废的订单

## 三、结算规则

### 支付方式
1. **结价支付**（现金）
   - 应收金额 = 金价 × 克重 + 工费总额
   - 必须录入当日金价

2. **实物抵扣**（以旧换新）
   - 客户提供黄金，抵扣原料成本
   - 客户只需支付工费部分
   - 需要称重记录客户提供的金重

## 四、退货规则

### 退货类型
1. **退给供应商**：商品部发现质量问题，退回供应商
2. **退给商品部**：展厅发现问题，退回商品部仓库

### 退货流程
1. 发起退货申请（填写克重、原因）
2. 等待审批（管理层或商品专员审批）
3. 审批通过后执行退货
4. 完成退货，扣减库存

### 退货原因分类
- 质量问题：工艺缺陷、材质问题
- 款式不符：与订单要求不符
- 数量差异：重量与入库单不符
- 工艺瑕疵：细节处理不到位
- 其他

### 注意事项
- 退货必须有充足库存才能执行
- 退货后会自动扣减对应位置的库存
- 退给供应商会更新供应商的供货统计

## 五、库存规则

### 多仓库管理
- 商品部仓库：入库商品默认存放位置
- 展厅：销售展示区域
- 在途：转移中的商品

### 库存转移
- 转移需要指定：商品名称、克重、发出位置、目标位置
- 发出位置必须有足够库存
- 接收方需要确认实际接收重量

## 六、供应商管理

### 供应商评级（建议）
- A级：月供货 > 1000g，退货率 < 2%
- B级：月供货 > 500g，退货率 < 5%
- C级：其他

### 统计指标
- 总供货金额（工费）
- 总供货重量
- 供货次数
- 退货次数（待实现）

### 角色说明

| 角色 | 主要职责 | 可访问页面 |
|------|----------|------------|
| 柜台 | 销售开单、接收库存 | 聊天、分仓库存、客户管理、退货管理 |
| 商品专员 | 入库管理、库存转移 | 聊天、分仓库存、供应商管理、退货管理 |
| 结算专员 | 结算审核、打印 | 聊天、结算管理 |
| 业务员 | 客户服务、销售查询 | 聊天、客户管理 |
| 财务 | 财务管理、对账 | 聊天、财务对账 |
| 管理层 | 全局管理、审批 | 所有页面 |

### 权限说明

1. **柜台（counter）**
   - 允许：创建销售单、接收库存、管理客户、退货给商品部
   - 禁止：商品入库、创建结算单、发起转移、管理供应商

2. **商品专员（product）**
   - 允许：商品入库、发起转移、管理供应商（不能删除）、退货给供应商
   - 禁止：创建销售单、创建结算单、接收库存、管理客户

3. **结算专员（settlement）**
   - 允许：创建结算单、确认结算、打印结算单
   - 禁止：商品入库、创建销售单、库存转移

4. **业务员（sales）**
   - 允许：查询销售单、管理客户
   - 禁止：所有写操作（入库、开单、转移等）

5. **财务（finance）**
   - 允许：财务对账、查看报表
   - 禁止：所有业务操作

6. **管理层（manager）**
   - 允许：所有功能，包括删除数据、数据分析、审批等

## 八、常见问题处理

### 库存不足
- 检查商品名称是否正确
- 确认位置是否正确
- 查看是否有在途转移

### 重复入库
- 系统会自动累加同名商品库存
- 如需区分批次，建议在商品名称中加入批次号

### 订单号查询
- 入库单：以 RK 开头（如 RK20260112001）
- 销售单：以 XS 开头（如 XS20260112001）
- 退货单：以 TH 开头（如 TH20260112001）
- 转移单：以 TR 开头（如 TR20260112001）
